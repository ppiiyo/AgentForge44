## 🔑 Внешние API-ключи провайдеров ИИ

### 1. Google Gemini API Key (Основной)
Этот ключ необходим для полноценной работы платформы, так как он обеспечивает работу узлов **Gemini Node**, автоматического ревьювера (**Reviewer Node**), оптимизатора промптов и ассистента-копилота.

* **Где взять:**
  1. Перейдите на платформу **[Google AI Studio](https://aistudio.google.com/)**.
  2. Войдите со своим Google-аккаунтом.
  3. Нажмите кнопку **"Get API Key"** в левом верхнем углу меню.
  4. Нажмите **"Create API Key"** и выберите проект (или создайте новый в один клик).
  5. Скопируйте созданный ключ (он начинается на `AIzaSy...`).
* **Как настроить:**
  Вставьте полученное значение в файл `.env` на сервере:
  ```env
  GEMINI_API_KEY=AIzaSyВашКлючСюда
  ```
  > 💡 **Режим Презентационной Песочницы:** Если вы хотите протестировать интерфейс без реального ключа, укажите специальное значение:
  > ```env
  > GEMINI_API_KEY=sandbox_free_test_gemini
  > ```
  > Платформа перейдет в режим симуляции и будет имитировать ответы языковых моделей абсолютно бесплатно.

---

### 2. OpenAI API Key
Используется, если в ваших агентных пайплайнах вы хотите отправлять запросы моделям семейства GPT (GPT-4o, GPT-3.5-Turbo).

* **Где взять:**
  1. Перейдите на портал разработчиков **[OpenAI Platform](https://platform.openai.com/)**.
  2. В боковом меню выберите раздел **"API Keys"** (или перейдите по прямой ссылке [platform.openai.com/api-keys](https://platform.openai.com/api-keys)).
  3. Нажмите **"Create new secret key"**, задайте имя ключу и сохраните его (начинается на `sk-...`).
  *Обратите внимание: для успешных запросов на балансе вашего аккаунта OpenAI должны быть средства.*
* **Как настроить:**
  ```env
  OPENAI_API_KEY=sk-proj-ВашКлючСюда
  ```

---

### 3. Anthropic Claude API Key
Требуется для подключения моделей Claude (Claude 3.5 Sonnet, Claude 3 Opus) в агентную сеть.

* **Где взять:**
  1. Зарегистрируйтесь на **[Anthropic Console](https://console.anthropic.com/)**.
  2. Откройте вкладку **"API Keys"**.
  3. Нажмите **"Create Key"** и скопируйте секретную строку (начинается на `sk-ant-...`).
* **Как настроить:**
  ```env
  ANTHROPIC_API_KEY=sk-ant-ВашКлючСюда
  ```

---

### 4. Локальный Ollama (Без ключей)
Позволяет запускать бесплатные локальные модели с открытым исходным кодом (Llama 3, Mistral, Gemma, Qwen) прямо на вашей видеокарте/процессоре, полностью конфиденциально.

* **Где взять и как запустить:**
  1. Скачайте и установите приложение с официального сайта **[Ollama](https://ollama.com/)**.
  2. Запустите Ollama в вашей системе.
  3. Скачайте нужную модель через терминал вашего компьютера. Например:
     ```bash
     ollama run llama3
     ```
  4. Ollama по умолчанию запустит локальный API-сервер на порту `11434`.
* **Как настроить:**
  Укажите адрес локального сервера Ollama в файле `.env`:
  ```env
  OLLAMA_HOST=http://localhost:11434
  ```

## 📦 API-ключи внешних баз знаний (RAG)

### 1. Pinecone, Weaviate, Qdrant
Используются для семантического поиска RAG по большим библиотекам ваших документов во внешних облачных векторных СУБД.

* **Pinecone:**
  - **Где получить:** Зарегистрируйтесь на **[Pinecone.io](https://www.pinecone.io/)**. Перейдите во вкладку **"API Keys"** и скопируйте ключ. Создайте индекс во вкладке **"Indexes"**.
  - **Конфигурация в `.env`:**
    ```env
    VECTOR_STORE_PROVIDER=pinecone
    VECTOR_STORE_API_KEY=ваш-api-ключ-pinecone
    VECTOR_STORE_ENV=us-east-1-gcp
    VECTOR_STORE_INDEX=my-rag-index
    ```

* **Weaviate / Qdrant:**
  Получите эндпоинт и ключ доступа в соответствующих консолях разработчиков и пропишите их:
  ```env
  VECTOR_STORE_PROVIDER=qdrant
  VECTOR_STORE_ENDPOINT=https://your-qdrant-cluster.cloud.qdrant.io:6333
  VECTOR_STORE_API_KEY=ваш-api-ключ-qdrant
  ```

### 2. Tavily Search API (Поиск в реальном времени)
Обеспечивает агентам возможность искать информацию в поисковых системах Google/Bing прямо в момент выполнения графа.

* **Где взять:**
  1. Зарегистрируйтесь на платформе **[Tavily](https://tavily.com/)**.
  2. На дашборде скопируйте сгенерированный API-ключ (начинается на `tvly-...`).
* **Как настроить:**
  ```env
  TAVILY_API_KEY=tvly-ВашКлючСюда
  ```

## ⚙️ Обязательные системные секреты бэкенда

Для обеспечения безопасности ваших данных и корректного запуска сервера вам **необходимо сгенерировать криптографические ключи** и прописать их в `.env`.

### 1. `JWT_SECRET`
Используется бэкендом для генерации и криптографической проверки токенов авторизации пользователей при входе в систему.
* **Где взять:** Сгенерируйте случайную строку длиной не менее 32 символов.
* **Быстрая генерация в терминале:**
  ```bash
  node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  ```
* **Пример в `.env`:**
  ```env
  JWT_SECRET=4f56a5e1c258d471b63e9c8a0f...
  ```

### 2. `ENCRYPTION_MASTER_KEY`
Все внешние API-ключи (например, которые пользователи могут вводить в интерфейсе) сохраняются в базу данных в зашифрованном виде с алгоритмом **AES-256-GCM**. Для этого шифрования бэкенду нужен этот мастер-ключ.
* **Где взять:** Сгенерируйте случайную шестнадцатеричную строку длиной 64 символа (32 байта).
* **Быстрая генерация в терминале:**
  ```bash
  node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  ```
* **Пример в `.env`:**
  ```env
  ENCRYPTION_MASTER_KEY=bc911854ea01d2c94bb507f308a0dfce...
  ```

### 3. `AGENTFORGE_API_KEY` (Защита эндпоинтов)
Защищает REST API вашего готового графа (`POST /api/execute`, `POST /api/graphs`) от несанкционированных вызовов извне сторонними лицами.
* **Как настроить:** Задайте произвольный сложный токен безопасности:
  ```env
  AGENTFORGE_API_KEY=af_key_prod_99f2b8a3
  ```
  При отправке HTTP-запросов к API KostromAi44, внешнее приложение должно передавать этот токен в заголовке авторизации:
  ```bash
  Authorization: Bearer af_key_prod_99f2b8a3
  ```

## 🖥️ Инструкция по запуску проекта

### Способ А: Запуск через Docker Compose (Рекомендуемый)
Этот способ запускает полностью готовую изолированную среду со всеми сервисами (бэкенд, фронтенд, СУБД) одной командой. Не требует установки Node.js на ваш хост.

#### Шаги запуска:
1. **Подготовьте ключи:** Скопируйте файл `.env.example` в новый файл `.env`:
   ```bash
   cp .env.example .env
   ```
2. **Заполните `.env`** вашими API-ключами и сгенерированными секретами (см. разделы выше).
3. **Запустите контейнеры:**
   ```bash
   docker-compose up --build
   ```
4. **Готово!** Сервис автоматически соберет проект, проведет миграции базы данных и запустит сервер.
   - 🌐 Откройте в браузере: **[http://localhost:3000](http://localhost:3000)**.
   - Чтобы остановить приложение, нажмите `Ctrl + C` в терминале или выполните `docker-compose down`.

---

### Способ Б: Локальный запуск через npm (Для разработки)
Этот метод отлично подходит для ручной отладки, локального тестирования и внесения изменений в код.

#### Требования:
- Установленная **Node.js v18** или новее.
- Установленный пакетный менеджер **npm** (идет в комплекте с Node.js).

#### Шаги запуска:
1. **Установите зависимости проекта:**
   ```bash
   npm install
   ```
2. **Создайте файл конфигурации окружения `.env`:**
   ```bash
   cp .env.example .env
   ```
3. **Заполните переменные окружения:** Откройте `.env` и впишите туда ваши API-ключи и сгенерированные секреты. Для быстрой автоматической генерации криптоключей на macOS/Linux можете запустить:
   ```bash
   echo "JWT_SECRET=$(openssl rand -base64 48)" >> .env
   echo "ENCRYPTION_MASTER_KEY=$(openssl rand -base64 48)" >> .env
   ```
4. **Запустите сервер разработки:**
   ```bash
   npm run dev
   ```
5. **Готово!** Консоль покажет лог запуска.
   - 🌐 Перейдите в браузере по адресу: **[http://localhost:3000](http://localhost:3000)**.

## 💡 Траблшутинг и частые вопросы

### 🛑 Ошибка: "GEMINI_API_KEY is not configured"
* **Причина:** Вы не заполнили `GEMINI_API_KEY` в файле `.env`, либо сервер не смог прочитать файл `.env` при запуске.
* **Решение:** Проверьте, что файл называется именно `.env` (без расширения `.txt`), находится в корневом каталоге проекта и содержит валидную строку `GEMINI_API_KEY=AIzaSy...`. Если у вас нет ключа, временно пропишите `sandbox_free_test_gemini` для включения бесплатного демо-режима.

### 🛑 Ошибка: "Master key is too short" / "JWT_SECRET is missing"
* **Причина:** Не установлены или слишком короткие системные криптоключи. Платформа требует их наличия для безопасности ваших данных.
* **Решение:** Сгенерируйте длинные строки (не менее 32 символов) для `JWT_SECRET` и `ENCRYPTION_MASTER_KEY` и добавьте их в `.env` (см. раздел [Системные секреты](#-обязательные-системные-секреты-бэкенда)).

### 🛑 Порт 3000 уже занят сторонней программой
* **Причина:** На вашем компьютере уже запущено другое веб-приложение, использующее порт `3000`.
* **Решение:** Вы можете временно остановить это приложение, либо в Docker-окружении переопределить порты в `docker-compose.yml`, изменив маппинг с `3000:3000` на `8080:3000` (в этом случае сайт будет доступен на `http://localhost:8080`).
