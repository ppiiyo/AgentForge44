# AgentForge44 Enterprise Applet: Manual Security Verification Rules

This reference document outlines the exact manual validation scenarios, test payloads, and expected secure behaviors designed to authenticate security postures for enterprise compliance.

---

## 1. SQL Injection (SQLi) Prevention Check
* **Objective**: Confirm that user inputs are safely escaped and never concatenated raw in SQL statements or database access queries.
* **Test Procedures**:
  1. Navigate to the marketplace, template creation, or custom action parameters.
  2. Input standard SQL escape and boolean termination injection strings in forms, search bars, or connection config fields:
     * `' OR '1'='1`
     * `'; DROP TABLE Webhooks; --`
     * `1 UNION SELECT username, password FROM users`
  3. Submit the form.
* **Expected Result**:
  * The system must treat the payload strictly as a literal, plain-text string.
  * No syntax or raw database schema errors are returned in responses.
  * Database adapters leverage parameterized query binding (ORM / Drizzle) which neutralizes input interpretation as code.

---

## 2. Cross-Site Scripting (XSS) Sanitization Check
* **Objective**: Ensure that user-entered code is escaped or removed to prevent execution in client browsers.
* **Test Procedures**:
  1. Locate the dynamic prompt node or system instruction fields in the workflow editor.
  2. Enter classic XSS payload tags or onload handler attributes:
     * `<script>alert(document.cookie)</script>`
     * `<img src="nonexistent" onerror="alert('XSS')">`
     * `javascript:alert(window.origin)`
  3. Attempt to trigger preview execution or share/load the workspace.
* **Expected Result**:
  * The raw tag brackets are automatically escaped safely using HTML entities (`&lt;` and `&gt;`).
  * Dangerous handler variables (`onerror`, `onload`, `onfocus`) are flagged, sanitizing raw content to safe placeholders or blocking event handlers.
  * The script never executes in the active browser segment.

---

## 3. Cross-Site Request Forgery (CSRF) Hardening Check
* **Objective**: Verify that state-changing API operations reject requests originating from external or unauthorized domains.
* **Test Procedures**:
  1. From an independent external browser context (or via tools like Postman/cURL), attempt to fire a `POST /api/execute` or `POST /api/graphs` request.
  2. Attempt to omit or poison the signature token payload (`X-CSRF-Token` or equivalent session cookie value).
* **Expected Result**:
  * The server rejects the state-changing action, returning `403 Forbidden` or `401 Unauthorized`.
  * The internal `CSRFProtector` signature check validates the cryptographic HMAC integrity of the token and blocks requests with invalid or missing session identifiers.

---

## 4. Server-Side Request Forgery (SSRF) Prevention Check
* **Objective**: Validate that outgoing HTTP requests generated by tools or integrations are restricted from accessing internal networks or metadata services.
* **Test Procedures**:
  1. Configure a tool node or dynamic integration with a loopback, private IP range, or local cloud metadata URL:
     * `http://127.0.0.1:3000`
     * `http://localhost:8080`
     * `http://169.254.169.254/latest/meta-data/`
     * `http://10.0.0.1/admin`
  2. Run the pipeline node.
* **Expected Result**:
  * Outgoing network components pre-validate targets via `ssrf-validator`.
  * The system immediately detects the private/loopback IP address and halts execution with `SSRF attempt blocked`.
  * No connection is ever established to internal backend endpoints.

---

## 5. Security & Authentication Bypass Check
* **Objective**: Prevent non-authenticated clients from accessing protected core APIs and endpoints.
* **Test Procedures**:
  1. Attempt to execute authenticated endpoints directly without setting valid credentials:
     * `GET /api/metrics/summary`
     * `DELETE /api/graphs/123`
  2. Send requests with empty, corrupted, or forged token structures:
     * `Authorization: Bearer invalid-token`
* **Expected Result**:
  * The system returns `401 Unauthorized` or `403 Forbidden` error codes.
  * No system analytics, telemetry, or workflow details are leaked.
